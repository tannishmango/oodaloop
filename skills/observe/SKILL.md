## Trigger

`/oodaloop-observe` or entering Observe phase.

## Workflow

1. Read current `STATE.md` for context.
2. Explore codebase using researcher agent (readonly).
3. Gather requirements from user input, existing docs, and code.
4. Structure findings into observations.
5. Update `PROJECT.md` with requirements and constraints.
6. Update `STATE.md` phase.

## Output

Structured observations in `PROJECT.md`, updated `STATE.md`.
